# Duru-Conductor — 완전 초보자를 위한 안내서

> 이 문서는 **컴퓨터 프로그래밍, 소프트웨어 개발, 프로젝트 관리를 한 번도 해본 적 없는 분**을 위해 쓰였습니다.
> 대화를 할 수 있으면 Duru-Conductor를 사용할 수 있습니다. 그것만으로 충분합니다.

---

## 이게 뭔가요?

무언가를 만들고 싶다고 상상해 보세요. 우리 가게 홈페이지, 사업 제안서, 어린이 그림책, 앱 — 뭐든요.

보통이라면 전문가 팀이 필요합니다. 디자이너, 개발자, 기획자, 프로젝트 매니저. 그러려면 시간, 돈, 인맥이 필요하죠.

**Duru-Conductor는 이것을 바꿉니다.** AI 비서가 여러분의 프로젝트 팀 전체 역할을 합니다. 무엇을 만들고 싶은지 말하면, Duru-Conductor가 첫 아이디어부터 완성품까지 모든 단계를 안내합니다.

- 코딩을 몰라도 됩니다
- "아키텍처"가 뭔지 몰라도 됩니다
- **무엇을 원하는지**만 알면 됩니다. 대략적으로라도요

나머지는 Duru-Conductor가 알아서 합니다.

---

## 어떻게 작동하나요?

Duru-Conductor는 네 단계를 따릅니다. 요리 레시피처럼요:

```
1단계: 접수 (Intake)     →  "뭘 만들고 싶으세요?"
2단계: 기획 (Planning)    →  "이렇게 만들겠습니다."
3단계: 실행 (Execute)     →  "한 단계씩 만들어 봅시다."
4단계: 회고 (Retrospective) →  "무엇을 배웠나요?"
```

각 단계마다 Duru-Conductor가 질문을 하고, **여러분이 "좋아요"라고 할 때까지 다음으로 넘어가지 않습니다.** 항상 여러분이 주도권을 갖습니다.

## 첫 번째 프로젝트 — 실제 예시

광명시에서 미용실을 운영하고 있고, 고객이 서비스와 가격을 보고 예약할 수 있는 간단한 홈페이지를 만들고 싶다고 해 봅시다.

### 1단계: 접수 — "뭘 원하시는지 알려주세요"

이렇게 입력합니다:

```
/duru-conductor
미용실 홈페이지를 만들고 싶어요. 고객이 서비스와 가격을 보고
온라인으로 예약할 수 있으면 좋겠어요.
```

Duru-Conductor가 요청을 읽고, 이미 알고 있는 것과 더 알아야 할 것을 파악합니다. 이런 질문을 할 수 있어요:

```
몇 가지 질문이 있습니다:

1. 미용실 이름이 뭔가요?
2. 고객이 온라인 결제도 해야 하나요, 아니면 시간 예약만 하면 되나요?
3. 호스팅 예산이나 선호가 있나요? [기본: GitHub Pages 무료 호스팅]
4. 한국어만 지원하나요, 아니면 한국어+영어 둘 다인가요?
```

주목하세요:
- **절대 5개 넘게 질문하지 않습니다.** 부담을 주지 않으려고요.
- **기본값을 제안합니다.** 모르겠으면 "기본값으로 해주세요"라고 하면 됩니다.
- **쉬운 말을 씁니다.** 전문 용어 없이.

대답하면 Duru-Conductor가 정리된 요약표를 보여줍니다:

```
| 항목        | 결정 사항                                    |
|------------|---------------------------------------------|
| 목표        | 서비스 목록 + 예약 기능이 있는 미용실 홈페이지    |
| 기술 스택    | HTML + CSS + JavaScript + Google Sheets 백엔드 |
| 플랫폼      | GitHub Pages (무료 호스팅)                     |
| 대상 고객    | 지역 고객, 한국어 사용자                        |
| 연동        | Google Sheets로 예약 데이터 관리                |
| 제약 조건    | 무료, 이번 주 안에 완성                         |
| 품질 기준    | 모바일 친화적, 스팸 방지                        |
```

Duru-Conductor가 묻습니다: **"이대로 진행할까요?"** "네"라고 하면 이 내용이 `INTAKE.md`라는 파일로 저장됩니다 — 합의한 내용의 기록이죠.

### 2단계: 기획 — "계획을 세웠습니다"

Duru-Conductor가 가상 팀을 자동으로 구성합니다:

```
| 역할          | 담당 업무                    |
|--------------|----------------------------|
| 설계자        | 웹사이트 구조 설계            |
| 프런트엔드 개발 | 페이지 제작                  |
| 비주얼 디자이너 | 전문적인 외관 만들기          |
| QA 엔지니어   | 모든 것이 작동하는지 테스트     |
```

그 다음 작업을 "워크스트림"으로 나눕니다 — 병렬로 진행할 수 있는 작업 묶음이죠:

```
| # | 워크스트림   | 산출물                   | 선행 조건 |
|---|-----------|-------------------------|---------|
| 1 | 디자인     | 레이아웃, 색상, 폰트 선정   | —       |
| 2 | 서비스 페이지 | 서비스 + 가격 안내 페이지  | #1      |
| 3 | 예약 시스템  | 예약 양식 + 백엔드        | #1      |
| 4 | 테스트     | 모바일+PC 전체 테스트      | #2, #3  |
```

그리고 특별한 것을 하나 보여줍니다 — **노력 비교표**:

```
| 워크스트림   | 사람 팀     | AI 활용 시   | 압축 비율 |
|-----------|-----------|------------|---------|
| 디자인     | 3일        | 20분        | 216배   |
| 서비스 페이지 | 2일       | 15분        | 192배   |
| 예약 시스템  | 5일       | 30분        | 240배   |
| 테스트     | 1일        | 10분        | 144배   |
| **합계**   | **11일**   | **75분**    | **211배** |
```

이것은 마법이 아닙니다 — AI가 반복적인 작업을 처리할 때 일어나는 일입니다. 이 표가 보여주는 것은: **완전하고 높은 품질의 버전을 만드는 것이 가치가 있다**는 것입니다. 비용이 몇 주가 아니라 몇 분이니까요.

Duru-Conductor가 묻습니다: **"이 계획대로 진행할까요?"** "네"라고 합니다.

### 3단계: 실행 — "만들어 봅시다"

Duru-Conductor가 순서대로 각 워크스트림을 진행합니다. 하나가 끝날 때마다 알려줍니다:

```
✅ 워크스트림 #1 (디자인) — 완료
   생성: styles.css, 레이아웃 템플릿
   다음: 서비스 페이지 제작

✅ 워크스트림 #2 (서비스 페이지) — 완료
   생성: index.html, services.html
   다음: 예약 시스템 제작
```

각 워크스트림이 끝날 때마다 `BUILD-LOG.md`에 진행 상황을 저장합니다. 이것은 **컴퓨터가 꺼지거나 창을 닫아도, 다음에 Duru-Conductor가 이어서 할 수 있다**는 뜻입니다.

마지막 워크스트림이 끝나면 이렇게 말합니다: "모든 작업이 완료되었습니다. 마무리 전에 `/review`로 품질 검사를 하는 것을 추천합니다."

### 4단계: 회고 — "무엇을 배웠나요?"

Duru-Conductor가 `RETROSPECTIVE.md`를 만들어서 정리합니다:
- 무엇을 만들었고 누구를 위한 것인지
- 잘된 점
- 개선할 점
- 배운 것
- 지표 (생성된 파일 수, 소요 시간 등)

이것은 여러분의 기록입니다. 다음 달에 또 다른 홈페이지를 만들 때, 여러분과 Duru-Conductor가 이걸 보고 더 잘할 수 있습니다.

## 여섯 가지 스킬 — 여러분의 도구 상자

Duru-Conductor가 메인 진행자이고, 다섯 개의 동반 스킬이 있습니다. 필요할 때 부르는 전문가라고 생각하세요.

### `/duru-conductor` — 프로젝트 매니저

새 프로젝트를 시작할 때의 출발점입니다. 접수, 기획, 실행, 회고까지 전체 주기를 관리합니다.

**언제 쓰나요:** "뭔가 만들고 싶어요" 또는 "프로젝트 아이디어가 있어요"라고 할 때.

### `/careful` — 안전망

위험한 명령을 실수로 실행하는 것을 막아줍니다. 중요한 것을 삭제하려고 하면, 실행 전에 위험을 쉬운 말로 설명해 줍니다.

**언제 쓰나요:** 파일, 데이터베이스, git을 다루는 모든 작업 시작 시에 활성화하세요. 초보자에게 특히 중요합니다.

**예시:**
```
⚠️ 높음 — 이 명령에 주의가 필요합니다

이 명령이 하는 일: "src/" 폴더와 그 안의 모든 것을 영구적으로 삭제합니다.
왜 위험한가: 휴지통이 없습니다. 한번 삭제하면 복구할 수 없습니다.
더 안전한 대안: 먼저 "mv src/ src-backup/"으로 이동해서 필요하면 복원할 수 있게 하세요.

진행하시겠습니까? (예 / 아니오)
```

**왜** 위험한지 가르쳐 줍니다. 단순히 막는 게 아니라. 시간이 지나면 이런 패턴을 스스로 인식하게 됩니다.

### `/review` — 품질 검사관

완성된 작업을 검사하고 구조화된 보고서를 만듭니다. 모든 프로젝트 유형에 작동합니다:
- **소프트웨어:** 보안 문제, 버그, 테스트 커버리지 확인
- **콘텐츠:** 일관성, 정확성, 완성도 확인
- **비즈니스:** 실현 가능성, 논리적 일관성, 완전성 확인

**언제 쓰나요:** 프로젝트를 완성한 후, 세상에 공개하기 전에.

### `/retro` — 성찰 코치

회고를 안내합니다 — 무슨 일이 있었는지, 무엇을 배웠는지, 다음에 무엇을 다르게 할지를 구조적으로 성찰하는 것입니다. 실제 지표(변경된 파일 수, 통과한 테스트 수, 소요 시간)를 수집해서 사실에 기반한 성찰을 합니다.

**언제 쓰나요:** 프로젝트나 중요한 작업을 마친 후에.

### `/azure-best-practices` — Azure 클라우드 전문가

Microsoft Azure에 배포할 때의 프로덕션 준비 규칙입니다. Azure 클라우드를 사용하는 프로젝트에만 해당합니다.

### `/m365-workflows` — Microsoft 365 전문가

Teams, SharePoint, Outlook 연동 패턴입니다. Microsoft 365에 연결하는 프로젝트에만 해당합니다.

## 스킬이 어떻게 함께 작동하나요?

스킬은 체인처럼 연결되도록 설계되었습니다:

```
/duru-conductor  →  프로젝트 기획 및 제작
     ↓
/careful    →  제작 중 실수 방지 (duru-conductor와 함께 활성화)
     ↓
/review     →  완성된 작업의 품질 검사
     ↓
/retro      →  배운 것 성찰
```

전부 쓸 필요는 없습니다. `/duru-conductor`만으로도 아이디어에서 완성품까지 갈 수 있습니다. 나머지는 깊이를 더합니다:

- **처음 무언가를 만드나요?** → `/duru-conductor` + `/careful`
- **만든 것의 품질을 확인하고 싶나요?** → `/review`
- **다음에 더 잘하고 싶나요?** → `/retro`
- **Azure에 배포하나요?** → `/azure-best-practices` 추가
- **Teams나 SharePoint 연동하나요?** → `/m365-workflows` 추가

## 완전 초보자를 위한 핵심 개념

### "스킬이 뭔가요?"

스킬은 AI에게 어떻게 행동할지 알려주는 지침입니다. `/duru-conductor`를 입력하면 AI가 프로젝트 매니저로 변합니다. `/review`를 입력하면 품질 검사관으로 변합니다. 같은 AI, 다른 전문성.

### "아티팩트가 뭔가요?"

아티팩트는 Duru-Conductor가 작업하면서 만드는 파일입니다. 기록이라고 생각하세요:

| 파일 | 뜻 | 쉽게 말하면 |
|---|---|---|
| `INTAKE.md` | 접수 요약 | "우리가 만들기로 한 것" |
| `ARCHITECTURE.md` | 기술 설계 | "이것의 구조" |
| `PLAN.md` | 실행 계획 | "단계별 할 일 목록" |
| `BUILD-LOG.md` | 진행 기록 | "지금까지 한 것" |
| `REVIEW.md` | 품질 보고서 | "잘한 것과 고칠 것" |
| `RETROSPECTIVE.md` | 배운 것 | "이 경험에서 얻은 교훈" |

이 파일들은 두 가지 역할을 합니다:
1. **프로젝트의 기억입니다.** 다음 주에 돌아오면 Duru-Conductor가 이것을 읽고 어디까지 했는지 압니다.
2. **스킬을 연결합니다.** `/review`가 `PLAN.md`를 읽고 계획한 것을 다 만들었는지 확인합니다. `/retro`가 `BUILD-LOG.md`를 읽고 실제로 무엇을 했는지 압니다.

### "잘못 말하면 어떻게 되나요?"

아무것도 안 망가집니다. Conductor는 중요한 일을 하기 전에 항상 확인합니다. 뭔가 잘못되면:
- "잠깐요" 또는 "멈춰주세요"라고 하면 됩니다
- "계획으로 돌아가요"라고 하면 됩니다
- Conductor는 묻지 않고 여러분의 작업을 절대 삭제하지 않습니다 (`/careful`이 활성화되어 있으면 위험을 먼저 설명합니다)

### "뭔가 이해가 안 되면요?"

말하세요! "그게 무슨 뜻이에요?"는 어떤 시점에서든 완전히 괜찮은 대답입니다. Duru-Conductor가 더 쉬운 말로 설명할 것입니다. 초보자를 위해 설계되었으니까 — 귀찮게 하는 게 아닙니다.

### "뭔가 설치해야 하나요?"

스킬을 지원하는 AI 코딩 에이전트가 필요합니다. 가장 일반적인 것들:
- **Claude Code** (추천) — `/duru-conductor`를 바로 입력
- **GitHub Copilot** — AGENTS.md 파일로 작동
- **Cursor** — AGENTS.md 파일로 작동

프로그래밍 언어, 데이터베이스, 프레임워크를 미리 설치할 필요 없습니다. Duru-Conductor가 만드는 과정에서 알아서 합니다.

### 설치하기

```bash
npx skills add parandurume-labs/duru-skills
```

이미 설치했는데 업데이트하고 싶으면, 같은 명령을 다시 실행하세요:

```bash
npx skills add parandurume-labs/duru-skills
```

## 다양한 상황 예시

### "우리 가게 홈페이지를 만들고 싶어요"

```
/duru-conductor
카페 홈페이지를 만들고 싶어요. 고객이 메뉴, 위치,
영업시간을 볼 수 있으면 좋겠어요. 간단한 문의 양식도요.
```

Duru-Conductor가 소상공인(SME) 프로젝트임을 감지하고, 한국 소상공인에 최적화된 템플릿을 자동으로 로드합니다 — 개인정보 보호 정책 안내, 스팸 방지, 모바일 우선 디자인 포함.

### "책을 쓰고 싶어요"

```
/duru-conductor
동네를 탐험하는 호기심 많은 고양이에 대한 어린이 그림책을
쓰고 싶어요. 대상 연령은 5-8세, 한국 시장이에요.
```

Duru-Conductor가 콘텐츠 팀(작가, 편집자, 비주얼 디자이너)을 구성하고 ARCHITECTURE.md 대신 OUTLINE.md를 만듭니다.

### "사업 제안서가 필요해요"

```
/duru-conductor
광명시 새로운 배달 서비스에 대한 사업 제안서가 필요해요.
잠재 투자자를 위한 발표 자료예요.
```

Duru-Conductor가 비즈니스 팀(비즈니스 분석가, 전략가, 작가)을 구성하고 재무, 타임라인, 성공 지표가 포함된 PLAN.md를 만듭니다.

### "뭔가 망가뜨린 것 같아서 무서워요"

```
/careful
```

`/careful` 모드를 활성화하세요. 이제 모든 위험한 명령이 실행 전에 설명됩니다. 실수로 프로젝트를 삭제할 수 없습니다.

### "만든 건데 괜찮은 건가요?"

```
/review
```

Review가 만든 모든 것을 스캔하고, 심각도별로 분류된 문제를 찾아서, 뭘 먼저 고쳐야 하는지 정확히 알려줍니다.

### "이 프로젝트에서 뭘 배웠지?"

```
/retro
```

Retro가 지표를 모으고, 성찰 질문을 하고, 나중에 돌아볼 수 있는 문서를 만듭니다.

## 기존 conductor에서 업그레이드하기

이전에 `parandurume-labs/conductor`를 설치한 경우, 아래 단계를 따라 `duru-skills`로 업그레이드하세요.

#### 1단계: 기존 conductor 스킬 삭제

```bash
# macOS/Linux:
rm -rf .agents/skills/conductor .agents/skills/azure-best-practices \
       .agents/skills/m365-workflows .agents/skills/careful \
       .agents/skills/review .agents/skills/retro .agents/skills/web-browser-review

# Windows (PowerShell):
Remove-Item -Recurse -Force .agents\skills\conductor, `
  .agents\skills\azure-best-practices, .agents\skills\m365-workflows, `
  .agents\skills\careful, .agents\skills\review, .agents\skills\retro, `
  .agents\skills\web-browser-review
```

#### 2단계: duru-skills 설치

```bash
npx skills add parandurume-labs/duru-skills
```

기존 7개 스킬 + 새로운 4개 스킬 (azure-ai-foundry, azure-security-audit, m365-copilot-extensions, korean-compliance) 총 11개가 설치됩니다.

#### 3단계: 명령어 변경

| 이전 명령어 | 새 명령어 |
|---|---|
| `/conductor` | `/duru-conductor` |

다른 스킬 명령어 (`/careful`, `/review`, `/retro` 등)는 그대로 사용합니다.

#### 4단계: 확인

AI 코딩 에이전트에서 `/duru-conductor`를 입력하세요. 접수 단계가 시작되면 성공입니다.

> **참고:** 기존 프로젝트 파일 (`INTAKE.md`, `ARCHITECTURE.md`, `BUILD-LOG.md`, `RETROSPECTIVE.md`)은 그대로 호환됩니다.

## Duru-Conductor를 최대한 활용하는 팁

1. **모르는 것에 솔직하세요.** "모르겠어요"는 훌륭한 대답입니다 — Duru-Conductor가 합리적인 기본값을 선택하고 넘어갑니다.

2. **노력 비교표를 읽으세요.** 완전하고 높은 품질의 버전을 만드는 것이 왜 가치 있는지 보여줍니다. AI가 꼼꼼함을 저렴하게 만들어 주니까요.

3. **불안하면 `/careful`을 쓰세요.** 안전망에 부끄러울 것은 없습니다. 경험 많은 개발자도 씁니다.

4. **공유하기 전에 `/review`를 돌리세요.** 여러분이 눈치채지 못할 것들을 잡아줍니다.

5. **매 프로젝트 후에 `/retro`를 돌리세요.** 성찰의 습관은 복리로 쌓입니다. 다섯 번째 프로젝트는 첫 번째보다 극적으로 나아집니다.

6. **과정을 신뢰하세요.** Conductor는 매 주요 단계에서 확인을 구합니다. 뭔가 이상하게 느껴지면 말하세요. 항상 되돌아갈 수 있습니다.

7. **아티팩트를 보관하세요.** INTAKE.md, PLAN.md, BUILD-LOG.md, REVIEW.md, RETROSPECTIVE.md — 이것이 프로젝트의 이야기입니다. Duru-Conductor가 이어서 작업하는 데 도움이 되고, 무엇을 배웠는지 기억하는 데 도움이 됩니다.

## 감사의 말

duru-skills v2.0의 여러 기능은 Garry Tan (Y Combinator 대표)의 **[gstack](https://github.com/garrytan/gstack)**에서 영감을 받았습니다. gstack은 잘 설계된 AI 스킬 컬렉션이 한 명의 개발자를 가상 엔지니어링 팀으로 변환할 수 있음을 보여주었습니다. 우리는 그 아이디어를 우리의 목표에 맞게 적용했습니다: 초보자 친화적, 제로 디펜던시, 소프트웨어뿐 아니라 모든 유형의 프로젝트 지원.
